思路

顺序的

流程示意图:

简单来说, 顺序的就是跟字面意思一样, 上右下左的一条一条地把边裁剪出来

逆序的

流程示意图略

逆序的与顺序的不同, 顺序的 keyframes, 直接从 0%到 100%, 就是整个绕边的效果, 而逆序的 keyframes 在 animation 执行的时候, 会加上 reverse 属性, 也就是执行顺序是从 100%到 0%, 来实现绕边的效果的;

小坑

在 animation 的动画变换中, 使用 clip-path 的 polygon 是要点点对应的

也就是说如果上面是四个坐标点, 下面也必须是四个坐标点, 就像是 keyframes success 一样, 如果像 keyframes fail 一样的话就会发生形变(但如果你就是要求形变的话就没关系)
