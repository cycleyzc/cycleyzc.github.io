云开发

微信小程序云开发

微信小程序的云开发是个很方便的功能, 云服务器有自带的数据库,存储功能和函数功能, 虽然现在(2019-08-29)还不是太稳定(前段时间服务器就挂了几个小时), 但是确实方便了开发者的开发流程

云函数

因为云函数有自带鉴权的特性, 所以在以前一些需要与后台接口配合才能取得到的数据, 现在可以直接使用云函数来取得数据, 例如获取用户信息, 获取微信绑定手机号等等

刚好最近做了获取用户信息和获取手机号的云函数功能, 就记录一下

获取用户信息

我觉得小程序开发者调用获取用户信息的api, 大部分不是为了获取用户微信昵称和用户头像的, 而是想拿到同一开放平台下那个代表用户唯一身份的unionid, 可是直接调用api是不会返回unionid的, 所以才要去要后端配合去解密敏感数据; 如果微信官方能提供直接获取unionid的api, 这当然是最好, 可是现在是没有的(截止至2019-08-29, 是没有不需要条件的直接获取unionid的api, wx.login能获取到unionid, 但是规则以及限制更多)

不使用云函数的流程(使用button, open-type为getUserInfo):
获取用户信息(wx.getUserInfo) -> encryptedData和iv抛到后台调用解密解析 -> 返回完整的用户信息(包含unionid)

使用云函数的流程(使用button, open-type为getUserInfo):
获取用户信息(wx.getUserInfo) -> cloudID抛到云函数解析 -> 返回完整的用户信息(包含unionid)

就看以上两个流程, 长度没什么不同, 不同就在流程涉及人员而已, 一个需要前端后端配合, 一个只需要前端即可(如果是个全栈工程师, 就当我没说…)

获取手机号

不使用云函数流程(使用button, open-type为getPhoneNumber):
获取手机号 -> encryptedData和iv抛到后台调用解密解析 -> 返回完整的手机号信息

使用云函数流程(使用button, open-type为getPhoneNumber):
获取手机号 -> cloudID抛到云函数解析 -> 返回完整的手机号信息
